# 全量阅片策略详细指南

本文件包含 DICOM Doctor skill 的详细阅片执行策略。当宿主 AI 开始执行阅片时，请参考此指南。

## 为什么必须全量阅片

医学影像分析中，微小病灶（如 2-6mm 的肺结节）可能仅出现在 1-3 层切片中。以一套 800 层 CT（1mm 层距）为例：

- 一个 3mm 结节仅出现在约 3 层上
- 若每隔 40 层仅看 1 张（共 20 张），漏诊概率超过 96%
- 磨玻璃结节（GGO）在常规肺窗上可能完全不可见，只有在 GGO 专用窗上才能检出

因此，跳过任何一层都可能导致漏诊。这不是效率问题，而是患者安全问题。

## 推荐的高效全量阅片策略（三阶段法）

### 第一阶段：快速全量扫描

1. 获取全部 PNG 文件列表（按文件名自然排序）
2. **GGO 窗优先**：对每批切片，先独立检视 GGO 窗（`_ggo.png`，WW=800, WL=-500）
   - 磨玻璃结节可能**仅**在 GGO 窗上可见
   - 在 GGO 窗上发现但肺窗未见的淡薄影，报告为"疑似磨玻璃结节（仅 GGO 窗可见）"
3. **肺窗轮**：每批读取 10-20 张肺窗图片，重点关注肺结节、磨玻璃影、肺纹理
4. **纵隔窗轮**：用对应的 `_mediastinum.png` 再扫一遍，重点关注纵隔结构、淋巴结、心脏、大血管、胸腔积液
5. **双侧检查**：每张图片都要同时检查左肺和右肺，左肺舌段(S4/S5)是 GGO 结节高发区，最容易遗漏
6. 重点标记"疑似异常"的层面，记录其文件名
7. **反保守偏见**：≥2mm 的圆形高密度影应优先报告为疑似结节，而非轻易归为"血管断面"

### 第二阶段：精细复查异常区域

1. 对所有被标记为"异常"或"疑似异常"的层面，调取其前后 5-10 层的图片
2. 仔细分析异常征象的连续性、形态和密度特征
3. 准确测量结节大小、判断分级分类（如 Lung-RADS、LI-RADS）

### 第三阶段：结果回填与报告生成

1. 将每张图片的分析结果通过 `parse_ai_response()` 回填到 `ReviewResult` 中
2. 确保每张图片都有明确的结论（正常/异常/无法识别），不要留下"待检视"状态
3. 全部分析完成后，生成 PDF 报告

## 禁止的操作

| 禁止行为 | 原因 |
|---------|------|
| Collage 拼图阅片 | 大幅降低每张切片分辨率，微小病灶（尤其是磨玻璃结节）完全丢失 |
| 抽样阅片（每隔 N 层看一张）| 3mm 结节仅出现在约 3 层中，抽样漏诊率 >96% |
| 缩放切片后再提交 | 降低分辨率导致微小病灶不可见 |
| 仅依赖 MIP 或常规肺窗 | GGO 结节可能仅在 GGO 窗上可见 |
| 以"层面显示正常"为由批量跳过 | 相邻层可能隐藏着早期磨玻璃结节 |
| 以"腹部层面"为由跳过低位层面 | 腹部层面可能扫及肝脏、肾上腺等重要结构 |

## 批次控制规范

- 每次循环最多向 AI 视觉模型提交 15-20 张原始分辨率的切片
- 将全部切片分成 40+ 个批次，依次进行推断
- 每个批次输出进度信息（如"批次 3/42，切片 31-45/832"）
- 全部批次推断完毕后，汇总全部疑似病灶，去重合并后生成最终报告
