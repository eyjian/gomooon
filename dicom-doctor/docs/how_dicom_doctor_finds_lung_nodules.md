# DICOM Doctor 如何找出肺结节？——CAD 算法 + 大模型视觉阅片 + 交叉验证的三层协作架构解析

> **作者**：DICOM Doctor 项目组  
> **版本**：基于 DICOM Doctor v2.10.0  
> **关键词**：肺结节检测、CAD、大模型阅片、DICOM、胸部CT、Lung-RADS

## 一、引言

肺结节的早期检出对肺癌筛查至关重要。一个 3mm 的肺结节在 800 层 CT（1mm 层距）中仅出现在约 3 层上——如果每隔 40 层看一张（共 20 张），漏诊概率超过 **96%**。这不是效率问题，是患者安全问题。

DICOM Doctor 是一个开源的 AI 辅助医学影像阅片工具（[GitHub](https://github.com/eyjian/ai-skills)），它采用 **「CAD 算法预检 + 大模型视觉阅片 + 交叉验证」三层协作架构**，在真实临床数据上实现了对 2mm 级微小结节的可靠检出。

本文将深入解析这三层架构的工作原理、各自的优劣势，以及它们如何配合互补来最大化检出率、最小化漏诊风险。

## 三、第一层：CAD 算法预检——从原始 HU 数据中"海选"候选结节

### 3.1 什么是 CAD？

CAD（Computer-Aided Detection，计算机辅助检测）是一个**纯算法模块**，不使用任何大模型或深度学习，完全基于经典的医学图像处理技术。它直接操作 DICOM 文件中的原始 HU（Hounsfield Unit）数据——这是 CT 扫描中每个体素的密度值，精确到整数级别。

> **HU 值参考**：空气 ≈ -1000，肺实质 ≈ -700~-500，磨玻璃结节（GGO）≈ -700~-400，实性结节 ≈ 20~200，骨骼 ≈ 400~1000+

### 3.2 算法流程（6 步）

CAD 的检测流程可以概括为以下 6 个步骤：

#### 步骤 1：读取 DICOM Volume

使用 SimpleITK 库将 DICOM 文件加载为 3D 体积数据（numpy 数组），同时获取体素间距（spacing）和原点（origin）信息。这些空间信息对于后续计算结节的物理尺寸至关重要。

```
CAD: 加载 volume (292, 512, 512), spacing=[0.6836, 0.6836, 1.25], HU=[-1024, 3071]
```

#### 步骤 2：2D 逐层肺分割

通过填充法获得完整的肺轮廓掩膜（lung_contour），将肺实质从胸壁、纵隔、脊柱等非肺组织中分离出来。后续所有检测都在这个肺轮廓内进行，避免将肋骨、脊柱等高密度结构误报为结节。

#### 步骤 3：密度阈值检测

在肺轮廓内，根据 HU 值范围提取两类候选区域：

| 候选类型 | HU 范围 | 检测目标 |
|---------|---------|---------|
| **实性候选** | -100 ~ 500 HU | 实性结节、钙化灶 |
| **GGO 候选** | -650 ~ -300 HU | 磨玻璃结节（最易漏诊的类型） |

对于厚层 CT（层距 ≥ 1mm），还会执行额外的形态学处理：
- **binary_closing**：填补微小结节因层距过大导致的体素间隙（带膨胀保护，防止过度膨胀）
- **逐层 2D 去噪**：对 GGO 掩膜进行去噪，去除每层小于 8 体素的碎片（这个阈值经过精心校准——≥10 会杀死真实 GGO 结节，≥8 是最优平衡点）

#### 步骤 4：3D 连通域标记 + 形态学特征计算

使用 `scipy.ndimage.label` 进行 3D 连通域分析，将相邻的高密度体素聚合成独立的候选区域。对每个候选计算以下形态学特征：

| 特征 | 说明 | 结节典型值 | 血管典型值 |
|------|------|-----------|-----------|
| 等效直径 | 体素体积换算的球形直径 | 2~15mm | 1~3mm |
| 球形度 | 接近球形的程度 | 0.6~1.0 | 0.2~0.5 |
| elongation | 长短轴比 | 1.0~1.5 | 1.5~3.0+ |
| z_slices | 跨越的层面数 | 2~10 | 1~2 |
| mean_HU | 平均密度 | 实性 50~200 / GGO -500~-400 | 变化大 |
| max_HU | 最大密度 | < 400 | 常 > 400 |

#### 步骤 4.5：空间邻近碎片二次聚合

距离 < 2mm 的小候选自动合并。这解决了因部分容积效应导致的同一结节被分裂为多个碎片的问题。

#### 步骤 5：多维度评分排序

这是 CAD 的核心创新——**不使用硬阈值过滤**（容易误杀真结节），而是用 7 个维度的综合评分来排序：

| 评分维度 | 实性权重 | GGO 权重 | 说明 |
|---------|---------|---------|------|
| 球形度 | 0.20 | 0.20 | 越接近球形，越像结节 |
| elongation | 0.20 | 0.15 | 越接近 1.0，越不像血管 |
| 大小 | 0.15 | 0.15 | 2~10mm 最可疑 |
| HU 值 | 0.10 | 0.12 | 在典型结节 HU 范围内得高分 |
| z 层数 | 0.10 | 0.13 | 跨越 2~5 层最可疑 |
| 密度一致性 | 0.05 | 0.05 | 内部密度均匀 |
| **血管排除** | **0.20** | **0.20** | 血管特征越明显，分数越低 |

此外，还有多种**假阳性惩罚机制**（基于 2 例 5 个真实结节的校准数据）：

- **肺外缘空间惩罚**：距图像中心越远，越可能是胸壁伪影
- **微小血管截面惩罚**：体素 ≤ 10 且 z_slices = 2 且直径 < 2.5mm → 强降权
- **血管模式惩罚**：(max_HU - mean_HU) > 200 且 mean_HU < 50 → 典型血管横截面
- **肺尖/肺底区域惩罚**：z < 5% 或 z > 95% → 部分容积效应高发区
- **GGO 过淡惩罚**：mean_HU < -520 → 接近空气，不太可能是真 GGO

#### 步骤 6：空间聚类合并 + Top-N 截取

将空间上相近的候选合并（实性 8mm，GGO 20mm），输出评分最高的 Top-30 实性候选和 Top-15 GGO 候选。

### 3.4 CAD 的优势与局限

| | 优势 | 局限 |
|---|------|------|
| **精度** | 直接操作原始 HU 数据，精确到每个体素的密度值 | 无法理解复杂形态（分叶、毛刺等） |
| **灵敏度** | 能发现大模型在 PNG 图片上可能看不到的微小密度差异 | 假阳性率较高（约 60~89%） |
| **一致性** | 算法确定性执行，不受"心情"影响 | 无法区分结节和血管截面的形态学差异 |
| **速度** | 全量 292 层 CT 处理仅需数秒 | 不能给出临床分级（Lung-RADS） |

## 四、第二层：大模型视觉阅片——像放射科医生一样"看图"

### 4.1 大模型的角色

大模型（如 Kimi、Claude、GPT-4o 等多模态视觉模型）在这里扮演的是**"放射科医生"**的角色——它真正地"看"PNG 图片，用视觉理解能力判断是否存在结节。

这与 CAD 的工作方式完全不同：CAD 操作的是数值矩阵（HU 数据），大模型看到的是渲染后的灰度图像。两者的信息来源互补。

### 4.2 多窗位策略

同一层 CT 数据会被渲染为多个窗位版本，每个窗位突出不同的组织特征：

| 窗位 | 窗宽/窗位 | 用途 | 优先级 |
|------|----------|------|--------|
| **高灵敏度 GGO 窗** | WW=400, WL=-550 | 极淡磨玻璃结节（**最高优先级**） | ⭐⭐⭐⭐⭐ |
| **GGO 窗** | WW=600, WL=-600 | 磨玻璃结节 | ⭐⭐⭐⭐ |
| **肺窗** | WW=1500, WL=-600 | 实性结节、肺纹理 | ⭐⭐⭐ |
| **纵隔窗** | WW=400, WL=40 | 纵隔结构、钙化验证 | ⭐⭐ |

> **为什么 GGO 窗优先？** 磨玻璃结节（GGO）是最容易漏诊的结节类型，其密度仅比正常肺实质略高。在标准肺窗下，GGO 可能完全不可见，只有在专门的 GGO 窗下才能显现。

### 4.3 Prompt 工程：把放射科教科书"注入"大模型

大模型的阅片能力很大程度上取决于 Prompt 的质量。DICOM Doctor 的 Prompt 模板（约 5000 字）融合了放射科教科书级别的方法论：

**四阶段检视法**（每张图片都必须执行）：
1. 高灵敏度 GGO 窗优先检视
2. GGO 窗检视
3. 肺窗精细检视
4. 纵隔窗验证

**微小结节检测六步法**（2-6mm）：
1. 网格扫查法：将图像分成 4×4=16 个网格区域，逐格检视
2. 密度对比法：与周围正常肺组织对比
3. 连续层面验证：同一结节应在相邻 2-3 个层面出现
4. 血管鉴别四要素：条状+分支+沿肺纹理+连续多层 = 血管；圆形+局限+不沿纹理 = 结节
5. 胸膜下重点检查
6. 双侧强制扫查

**反保守偏见铁律**：
> ≥2mm 的圆形/类圆形高密度影**必须优先报告为疑似结节**，禁止轻易归为"血管断面"。宁可多报 100 个假阳性，也不能漏掉 1 个真结节。

### 4.4 CAD Hint 注入：算法给大模型"划重点"

CAD 的候选结节信息会作为 **CAD hint** 注入到每层的 Prompt 中。这是两者配合的关键桥梁：

```
⚠️ CAD 自动预检在本层附近检测到以下候选结节，请重点验证：
- 实性候选 #6 (score=0.971): 位置(189,222), 直径3.1mm, HU=144, z=197-199
  请仔细检视该区域，判断是否为真实结节。
```

**v2.4.7 的分层优化**：并非每层都注入全部候选——只有 CAD 候选附近 ±5 层（"重点层"）才使用完整版 Prompt + 分层 CAD hint，远离候选的"快扫层"使用精简版 Prompt（约 600 字），节省约 50-70% 的 token 消耗。

### 4.5 大模型的输出

大模型对每张图片输出结构化 JSON：

```json
{
  "conclusion": "异常",
  "abnormality_desc": "右肺中叶内段(S5)可见约3mm×2mm实性结节",
  "confidence": "中",
  "details": "该层面右肺中叶内段可见一圆形高密度影...",
  "location": "右肺中叶内段(S5) (第197-199层)",
  "size_mm": "3x2",
  "lung_rads": "2类",
  "recommendation": "建议12个月低剂量CT随访",
  "bounding_boxes": [{"x": 0.35, "y": 0.42, "width": 0.05, "height": 0.04}]
}
```

### 4.6 大模型的优势与局限

| | 优势 | 局限 |
|---|------|------|
| **形态理解** | 能理解分叶、毛刺、晕征等复杂形态特征 | 受 PNG 图片分辨率和窗位设置影响 |
| **鉴别诊断** | 能区分结节、血管、伪影 | 可能漏掉极淡的 GGO（密度差异太小） |
| **临床分级** | 能给出 Lung-RADS 分级和随访建议 | 不同模型能力差异大 |
| **上下文理解** | 能结合相邻层面信息综合判断 | 上下文窗口有限，需分批处理 |

## 五、第三层：CAD vs AI 交叉验证——防止遗漏的"质控主任"

### 5.1 为什么需要交叉验证？

在真实案例中，我们发现了一个危险的漏诊模式：

> Case 1 中，CAD 检出了一个 score=0.96 的实性候选（右肺下叶前基底段 2mm），大模型在阅片描述中提到了"可疑微小结节"，但 conclusion 却标为"正常"——导致该结节未进入最终报告。而这个结节正是医院确诊的炎性肉芽肿。

这就是"描述了但不报"的漏诊模式——大模型看到了，但没有正式报告。交叉验证就是为了捕获这类遗漏。

### 5.2 交叉验证的工作原理

交叉验证在 AI 阅片完成后、PDF 报告生成前自动执行：

```
对每个 CAD 评分 ≥ 0.80 的高分候选：
│
├── 查找对应 z-range（±2 层容差）内的 AI 阅片结论
│
├── 情况 1: AI 报告了"异常"
│   └── ✅ 确认，无告警
│
├── 情况 2: AI 报告了"正常"，但描述中明确提及排除理由
│   │   （如"确认为血管分支""伪影""支气管"）
│   └── ✅ 已审查并排除，无告警
│
├── 情况 3: AI 报告了"正常"，描述中提及可疑征象
│   │   （如"可疑微小结节""不排除结节"）
│   └── ⚠️ 告警！类型: described_but_not_reported（描述未报）
│
├── 情况 4: AI 报告了"正常"，无任何提及
│   └── ⚠️ 告警！类型: missed_candidate（遗漏候选）
│
└── 情况 5: 该区域无 AI 阅片结果
    └── ⚠️ 告警！类型: no_coverage（无覆盖）
```

### 5.3 告警分级

| 告警类型 | 严重度 | 含义 |
|---------|--------|------|
| `missed_candidate` | **HIGH** | CAD 高分候选被 AI 完全忽略 |
| `described_but_not_reported` | **HIGH** | AI 描述中提到了可疑征象但结论标为正常 |
| `no_coverage` | **MEDIUM** | 候选区域内无 AI 实际审阅结果 |

### 5.4 告警在报告中的呈现

交叉验证的告警会以醒目的表格形式展示在 PDF 报告的专门章节中：

| 严重度 | 类型 | 候选 | 评分 | z-range | 说明 |
|--------|------|------|------|---------|------|
| HIGH | 遗漏候选 | SOLID#3 | 0.952 | 138-142 | CAD 评分 0.952 的实性候选在 z=138-142 范围内，AI 全部标为正常且未提及可疑征象 |

## 六、三层协作的完整流程

让我们用一个真实案例来串联整个流程：

### 案例：2026-01-10 胸部 CT（1.25mm 层厚，292 层）

**医院诊断**：4 个结节（1 个 GGO + 3 个实性），均为炎性改变/肉芽肿。

#### 第一层：CAD 预检

CAD 在数秒内扫描全部 292 层，输出：
- 30 个实性候选（Top-30）
- 15 个 GGO 候选（Top-15）
- 其中 4 个真实结节全部被检出，评分均 ≥ 0.90

但同时也有约 31 个假阳性（血管截面、伪影等），假阳性率约 60-89%。

#### 第二层：大模型阅片

大模型逐张检视全部 292 层 × 4 窗位 = 1168 张 PNG 图片：
- 在 CAD 候选附近的"重点层"，使用完整版 Prompt + CAD hint 重点验证
- 在远离候选的"快扫层"，使用精简版 Prompt 快速扫查
- 对每个 CAD 候选给出三选一结论：确认结节 ✅ / 可疑待定 ⚠️ / 明确排除 ❌
- 同时独立发现 CAD 可能遗漏的异常

#### 第三层：交叉验证

对比 CAD 和 AI 的结果：
- 4 个真实结节：CAD 检出 ✅ + AI 确认 ✅ → 无告警
- 31 个假阳性：CAD 检出但 AI 明确排除（给出血管/伪影理由）→ 无告警
- 如果 AI 遗漏了某个高分候选 → 发出告警，提示复查

#### 最终报告

生成医院风格的 PDF 报告，包含：
- 结节聚合汇总表（4 个独立结节，含位置/类型/大小/Lung-RADS 分级）
- 逐层异常明细（每个异常层面的详细数据）
- 异常影像展示（嵌入 PNG 图片 + 标注）
- 诊断意见和随访建议
- 交叉验证告警（如有）

## 七、三层架构的角色分工

| 角色 | 做了什么 | 类比 | 优势 | 劣势 |
|------|---------|------|------|------|
| **CAD 算法** | 从原始 HU 数据中用密度阈值 + 3D 连通域分析找出"可疑区域" | 自动化"初筛机器" | 灵敏度高、速度快、操作原始数据 | 假阳性多、不懂形态 |
| **大模型** | 逐张看 PNG 图片，用视觉理解能力判断是否真的是结节 | "放射科医生" | 理解形态、能分级、能鉴别 | 受图片质量影响、可能漏极淡 GGO |
| **交叉验证** | 对比两者结果，找出遗漏 | "质控主任" | 防止"描述了但不报"的漏诊 | 依赖前两层的输出质量 |

**核心设计哲学**：

> CAD 负责"海选"（高灵敏度初筛），大模型负责"精选"（视觉确认 + 分级），交叉验证负责"兜底"（防遗漏）。三者缺一不可——CAD 没有大模型的话假阳性太多，大模型没有 CAD 的话可能漏掉极微小的结节，没有交叉验证的话无法发现两者的盲区。

## 八、关键技术细节

### 8.1 为什么 CAD 不用深度学习？

DICOM Doctor 的 CAD 模块有意选择了传统算法而非深度学习，原因包括：

1. **可解释性**：每个候选的评分维度完全透明，可以精确解释为什么某个区域被标记为可疑
2. **零训练数据要求**：不需要大量标注数据，基于物理原理（HU 密度）和几何特征（球形度、elongation）工作
3. **可校准性**：评分参数可以基于少量真实案例精确调优（当前基于 2 例 5 个结节校准）
4. **轻量级**：仅依赖 SimpleITK + scipy + numpy，无需 GPU，数秒完成全量扫描

### 8.2 GGO 去噪的精细校准

GGO（磨玻璃结节）的检测是最大的技术挑战。以 Case 2 的真实 GGO 结节（GT1，4mm×3mm，HU=-409）为例：

| 去噪方法 | 连通域数 | GT1 存活？ | 体素数 |
|---------|---------|-----------|--------|
| 3D opening | - | ❌ 完全消失 | 0 |
| 2D opening (3×3) | - | ❌ 消失 | 0 |
| 逐层 2D 去噪 ≥ 3 | 24,464 | ✅ 存活 | 38 |
| 逐层 2D 去噪 ≥ 5 | 9,839 | ✅ 存活 | 27 |
| **逐层 2D 去噪 ≥ 8** | **3,717** | **✅ 存活** | **17** |
| 逐层 2D 去噪 ≥ 10 | 2,250 | ❌ 丢失 | 0 |

最终选择 **≥ 8 体素/层** 作为去噪阈值——这是在"减少噪声"和"保留真实 GGO"之间的最优平衡点。

### 8.3 上下文窗口优化（v2.4.7）

800 层 CT × 4 窗位 × 完整版 Prompt ≈ 数十 MB 的 token 消耗，远超任何大模型的上下文窗口。DICOM Doctor 通过以下策略解决：

1. **分批持久化**：每批 15-20 张，完成一批立即保存到磁盘，支持断点续跑
2. **双层 Prompt**：重点层用完整版（~5000 字），快扫层用精简版（~600 字）
3. **分层 CAD hint**：每层只注入附近 ±5 层的候选信息，而非全部候选

优化效果：prompt token 总量减少约 50-70%，manifest 文件从 11.6MB 降至 1.8MB。

## 九、版本演进：从 v2.0 到 v2.10 的假阳性优化之路

DICOM Doctor 的 CAD 模块经历了 8 个大版本的迭代优化，假阳性率从最初的 ~95% 逐步降低到 ~60%：

| 版本 | 核心改进 | 假阳性变化 |
|------|---------|-----------|
| v2.2 | binary_closing + 碎片聚合 + 多维度评分 | 建立评分体系 |
| v2.3 | 血管排除评分 + 微小血管截面惩罚 | 显著降低血管假阳性 |
| v2.4.1 | HU 评分收紧 + GGO elongation 惩罚 | 进一步区分真假 |
| v2.4.4 | mean_HU < 20 强降权 + 血管模式惩罚 + 肺尖/肺底惩罚 | Case 1 假阳性大幅下降 |
| v2.5.0 | 肺外缘空间惩罚 + 外缘+微小灶联合惩罚 + GGO HU 精细化 | Case 2 假阳性从 31→14 |
| v2.4.8 | CAD vs AI 交叉验证 | 新增质控层 |
| v2.10.0 | 「先试后判」视觉探测 + 移除环境变量误判 | 提升大模型协作可靠性 |

> **核心原则**：每次优化都必须通过回归验证——5 个真实结节的评分不能下降。"降低假阳性"和"不漏掉真结节"必须同时满足。

## 十、总结

DICOM Doctor 的肺结节检测不是单独靠 CAD 算法，也不是单独靠大模型，而是三层协作的结果：

1. **CAD 算法**从原始 HU 数据中以高灵敏度"海选"候选区域
2. **大模型**用视觉理解能力逐张"精选"确认，并给出临床分级
3. **交叉验证**对比两者结果"兜底"，防止任何一方的遗漏

这种架构的设计哲学是：**宁可多报一个可疑发现让医生复核，也不能漏掉一个可能的结节**。在真实临床数据上，该架构实现了对 2mm 级微小结节的零漏检（5/5 真实结节全部检出），同时通过持续的假阳性优化将误报率控制在可接受范围内。
