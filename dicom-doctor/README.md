# DICOM Doctor — AI 辅助医学影像处理 Skill

> **当前版本：v2.5.0**

---

## 简介

DICOM Doctor 是一个面向 AI 智能体的医学影像处理 Skill，提供两项核心能力：

- **DICOM 转 PNG**：支持**所有影像类型**（胸部CT、腹部CT、头颅MRI、腹部MRI、核磁等），自动识别影像类型并选择最佳窗位策略
- **AI 辅助阅片**：**当前版本仅支持胸部CT**，AI 逐张检视全部切片，自动检测肺结节/肿块/钙化，并生成医院风格的 PDF 检查报告

**您不需要懂编程，也不需要安装任何软件，只需和 AI 自然语言对话即可。**

---

## 什么是 Skill？

Skill 是一组预定义的专业指令、脚本和资源的集合，可以动态加载到 AI 平台中，让 AI 在特定任务上具备专业能力。DICOM Doctor Skill 加载后，AI 会自动按照专业的医学影像处理流程工作，无需您手动指导每一步。

## 功能详解

### 一、DICOM 转 PNG（支持所有影像类型）

将 DICOM 格式的医学影像文件转换为 PNG 图片，方便查看和分享。

| 影像类型 | 自动识别方式 | 窗位策略 |
|----------|-------------|---------|
| **胸部 CT** | Modality=CT + BodyPart=CHEST | 肺窗 / 纵隔窗 / GGO窗 / 高灵敏度GGO窗 / 骨窗 |
| **腹部 CT** | Modality=CT + BodyPart=ABDOMEN | 软组织窗 / 肝脏窗 / 骨窗 |
| **头颅 MRI** | Modality=MR + BodyPart=HEAD/BRAIN | T1 / T2 / FLAIR / DWI 自适应 |
| **腹部 MRI** | Modality=MR + BodyPart=ABDOMEN | T1 / T2 / DWI 自适应 |
| **通用** | 兜底（无法识别时） | 自动推断最佳窗位 |

**特色功能：**
- 🔍 **自动识别影像类型**：无需手动指定，AI 根据 DICOM 元数据自动判断
- 🖼️ **多窗位输出**：不同窗位突出不同组织结构，一次转换输出多套图片
- 📐 **低分辨率自动放大**：小尺寸图像自动 Lanczos 放大到至少 1024×1024

### 二、AI 辅助阅片（当前仅支持胸部 CT）

AI 逐张检视全部 CT 切片，检测异常并生成专业报告。

**检测能力：**
- 肺结节（实性结节、磨玻璃结节 GGO、部分实性结节）
- 肺部肿块
- 钙化灶
- 扫及区域异常（甲状腺、肝脏上段、肾上腺）

**阅片特点：**
- ✅ **全量逐张阅片**：每一张切片都会被检视，不抽样、不跳层，确保不遗漏微小病灶
- ✅ **多窗位联合检视**：高灵敏度GGO窗 → GGO窗 → 肺窗 → 纵隔窗，四窗联合提高检出率
- ✅ **CAD 计算机辅助检测**：内置 CAD 算法自动预筛候选区域，AI 逐一验证
- ✅ **CAD vs AI 交叉验证**：自动比对 CAD 预检结果与 AI 阅片结论，防止遗漏
- ✅ **Lung-RADS 分级**：按照国际标准对每个发现进行风险分级
- ✅ **反保守偏见**：宁可多报一个可疑发现让医生复核，也不遗漏潜在病灶

**可选增强功能：**
- 🔬 **超分辨率增强**：使用 Real-ESRGAN 技术放大图像（2倍/4倍），提升细节清晰度
- 📊 **MIP 最大密度投影重建**：连续多层投影，提高 2-6mm 微小肺结节检出率

### 三、PDF 报告生成

生成医院风格的 PDF 影像检查报告，内容包含：

| 报告章节 | 说明 |
|---------|------|
| 检查信息表 | 检查类型、日期、影像数量、窗口类型等基本信息 |
| AI 检视统计 | 总切片数、正常/异常/无法识别的数量统计 |
| 结节聚合汇总 | 将跨层面的异常合并为独立结节，含位置、类型、大小、分级 |
| 逐层异常明细 | 每个异常层面的详细数据，供精确定位复核 |
| 异常影像展示 | 嵌入异常切片图片 + 标注描述 + 分级分类 |
| 诊断意见 | 综合分析结论与随访建议 |
| 分级参考表 | Lung-RADS 完整分级说明 |
| 免责声明 | AI 辅助分析的法律声明 |

## 系统要求

### 模型要求

AI 辅助阅片功能**需要支持图像识别的多模态视觉模型**，以下模型已验证支持：

| 模型 | 厂商 | 阅片支持 | DICOM转PNG |
|------|------|---------|-----------|
| **Claude-4.6-Opus** | Anthropic | ✅ | ✅ |
| **Kimi-K2.5** | Moonshot AI | ✅ | ✅ |
| **GPT-5.4** | OpenAI | ✅ | ✅ |

> **注意**：纯文本模型（不支持图像识别的模型）仍可使用 DICOM 转 PNG 功能，但无法进行 AI 阅片。

### 运行环境

| 要求 | 说明 |
|------|------|
| Python | 3.8 及以上版本（AI 会自动检测，缺失时自动安装） |
| 依赖库 | pydicom、SimpleITK、Pillow、reportlab（AI 会自动安装） |
| 操作系统 | macOS / Linux / Windows 均支持 |

> 您无需手动安装任何依赖——AI 会自动检测环境并安装所需组件。

## 如何启用此 Skill

将 `SKILL.md` 文件添加到支持 Skill 的 AI 平台（如 Cursor、Claude Projects、Claude Code 等）中即可。

启用后，当您在对话中提到以下内容时，AI 会自动激活此 Skill：

- DICOM 文件、`.dcm` 文件、ZIP 影像包
- CT 扫描、MRI、核磁、核磁共振、胸部 CT、腹部 CT、头颅 MRI、腹部 MRI
- DICOM 转 PNG、影像转换、肺结节、医学影像、阅片、影像报告

## 使用步骤

### 第一步：提供影像文件

根据您的使用场景，有两种方式提供文件：

- **上传文件**（网页端 / 云端 AI）：将 DICOM 文件（`.dcm`）或包含多个 DICOM 文件的 ZIP 压缩包上传给 AI
- **指定本地路径**（本地智能体，如 Cursor / Claude Code 等）：直接告诉 AI 文件在本地的路径，无需上传

### 第二步：用自然语言描述需求

直接告诉 AI 您想要什么，例如：

> "帮我分析这个胸部 CT，看看有没有肺结节。"

> "这是我的胸部 CT 检查，请帮我生成一份影像报告。"

> "帮我把这个头颅 MRI 的 DICOM 文件转成 PNG 图片。"

> "这是腹部 CT 的 dcm 文件，帮我转换一下。"

### 第三步：获取结果

- **胸部 CT**：AI 完成全量阅片后，生成 PDF 格式的影像检查报告，并主动询问您是否需要对某个异常部分进行重点复核
- **其他影像类型**（MRI、腹部CT 等）：输出 PNG 转换结果

## 注意事项

### 关于阅片结果

- 🏥 **仅供参考**：AI 生成的报告不构成医学诊断，如有疑问请及时咨询专业医生
- 🎯 **宁多勿漏**：AI 采用"反保守偏见"策略，可疑发现会优先报告，最终判断权留给医生
- 📋 **Lung-RADS 分级**：报告中的分级仅作参考，临床决策需结合患者完整病史

### 关于影像质量

- 建议使用薄层 CT（层厚 ≤ 1.25mm）以获得最佳检出效果
- 厚层 CT（≥ 1mm）会自动启用额外的图像处理策略以提升微小结节检出率
- 如果某个窗位图像质量较差（如过曝），AI 会自动降级到备选窗位并在报告中注明

### 关于模型选择

- AI 阅片功能**必须使用多模态视觉模型**（如 Kimi-K2.5、Claude-4.6-Opus、GPT-5.4 等）
- 纯文本模型可以完成 DICOM 转 PNG，但无法进行阅片分析
- 不同模型的阅片精度可能存在差异，建议使用最新版本的大模型
